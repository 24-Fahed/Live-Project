# 后端开发任务书

## 一、部署架构调整

网关服务与后端服务部署于同一台服务器，采用单机部署模式。

| 服务 | 端口 |
|------|------|
| 网关 (Gateway) | 8080 |
| 后端 (Backend) | 8000 |

---

## 二、前端访问策略

### 2.1 微信小程序端

微信小程序通过网关代理访问后端服务。

### 2.2 管理员前端

管理员前端页面由网关静态托管，直接访问后端服务。由于存在跨域场景，后端需配置 CORS（跨域资源共享）策略。

---

## 三、网关职能修改

### 3.1 数据存储改造

遵循开闭原则，为前端数据存储层添加**全局单线程操作**能力，将现有实现改造为类 Redis 的内存数据存储模式。

### 3.2 WebSocket 延迟推送

设计 WebSocket 延迟推送机制：
- 网关侧进行数据变更的批量聚合
- 定时向后端批量同步更新
- 目的：减少后端调用频次，增大单次提交数据量，降低 MySQL 数据库压力

### 3.3 消息队列模拟

由数据存储服务控制实现轻量级消息队列，按设定时间间隔将数据变更打包推送至后端。
