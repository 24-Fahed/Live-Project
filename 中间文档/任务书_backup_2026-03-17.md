# 后端开发任务书

## 2. 跨域配置 (CORS) 匹配

当前 gateway 的 CORS 配置：

```javascript
app.use(cors({
    origin: '*',  // 允许所有来源
    methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS'],
    allowedHeaders: ['Content-Type', 'Authorization', 'X-Requested-With'],
    credentials: true
}));
```

**要求**：后端需要配置相同的 CORS 策略，否则前端请求会被浏览器拦截。

## 3. 鉴权逻辑匹配

当前系统的鉴权：

- 前端 `admin-api.js` 会在调用 `/api/v1/*` 接口时添加 `Authorization: Bearer <token>` 头
- 但后端没有强制鉴权中间件（所有接口目前是开放的）

**要求**：后端需要决定是否实现鉴权。如果要实现，需要与前端 token 逻辑保持一致。

## 4. 本地调试方式

前端通过 `frontend/config/server-mode.js` 中的 `API_BASE_URL` 变量配置后端地址。

### 步骤

#### 1. 实现后端

在 `backend/` 中实现后端，监听某个端口（如 8000），并实现所有前端需要的 API 路径。

#### 2. 修改前端配置

编辑 `frontend/config/server-mode.js`，修改最后一行，指向你的后端地址：

```javascript
// 原来（指向远程服务器）
export const API_BASE_URL = REAL_SERVER_URL;

// 改为（指向你的本地后端）
export const API_BASE_URL = 'http://localhost:8000';
```

或使用预设变量：
```javascript
export const API_BASE_URL = LOCAL_SERVER_URL;  // http://localhost:8080
```

#### 3. 运行前端项目

```bash
cd frontend
npm install
npm run dev:h5          # H5 模式（浏览器直接访问）
# 或
npm run dev:mp-weixin   # 微信小程序模式
```

#### 4. 调试

- 前端会调用配置的 `API_BASE_URL`
- 后端收到请求，处理并返回数据
- 通过浏览器开发者工具或后端日志调试

### 预设的服务器地址

| 变量 | 地址 | 用途 |
|------|------|------|
| `LOCAL_SERVER_URL` | `http://localhost:8080` | 本地调试 |
| `MIDDLEWARE_SERVER_URL` | `http://192.168.31.249:8081` | 中间层服务器 |
| `REAL_SERVER_URL` | `http://192.140.160.119:8000` | 真实后端 |
