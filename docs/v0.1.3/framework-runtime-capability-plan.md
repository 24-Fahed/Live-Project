### 6.3.1 配置盘点文档创建任务

在 `v0.1.3` 中，文档层除了继续维护网关配置说明、部署文档和架构文档外，还应正式新增一项任务：

- 创建“配置与配置命名盘点文档”

这份文档应至少覆盖：

- 当前所有关键环境变量清单
- 当前所有公开配置名称清单
- 当前哪些字段真实参与运行
- 当前哪些字段属于说明性或预留字段
- 当前哪些名称出于向后兼容原因暂时保留

这项任务的目标不是重复已有部署文档，而是为正式版治理建立一份专门的配置基线文档。

### 7.2.1 认证配置命名统一纳入下一阶段

当前认证基础设施中，实际被调用方使用的名称仍然是 `jwt_settings`。虽然从长期演进看，随着鉴权能力继续增加，这个名称后续需要升级为更准确的认证配置命名，但这项调整不放在当前版本处理。

在 `v0.1.3` 中，这项工作应被正式纳入修改计划，处理原则如下：

- 当前版本继续保留 `jwt_settings`，避免在联调阶段引入不必要的命名重构风险
- 下一阶段再统一认证配置对象、导入名称与文档术语，避免出现配置定义名和调用方名称不一致的问题
- 调整时应一次性完成配置层、调用层、文档层的同步收口，不再使用临时兼容命名

这项任务本质上属于“基础设施命名治理”，应与运行阶段能力和接入能力的框架化整理一起推进。
